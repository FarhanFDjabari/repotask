# Common Engineering Rules

- Prefer minimal, scoped changes.
- Do not refactor unrelated code.
- Follow the existing project architecture and conventions.
- Explain the root cause before proposing implementation.
- Include practical verification steps.
- Do not add dependencies without explicit approval.
- Do not create commits unless the user explicitly requests one.
- Never automate requirement approval, QA decisions, merge approval, conflict resolution,
  deployment approval, or release approval.
